Module Scopes

Leerdoelen

  • Global scopes
  • Block scopes

    • Javascript heeft twee scopes; global en local(blok) scopes.
    Elk variable declared buiten de functie behoort tot de gloabl scope en is overal in de code toegankelijk
    Elk functie heeft ook zijn eigen scope, dit noemt men "een local of een block scope". Elk variable dat in de function wordt geschreven behoort alleen tot de functie en is niet toegankelijk van buiten af.

    Opdracht 1

    Scopes geeft de context aan waar de variables gedeclareerd worden. Dit geeft de toegankelijkheid van de variables aan.
    Afhankelijk van de scope kan je dus toegang krijgen tot de waarde van de variable.

    Kijkend naar de volgende code:

    function printCar() { const carBrand = "Volvo"; } console.log(carBrand);

    Wat wordt er in de console gelogt? Wat voor scoping heb je hier te maken?

    Dit is eeen voorbeeld van block-scope.
    Dit zie je aan de plek waar const staat. Dit is in de functie toegankelijk en niet buiten de functie.
    Je krijgt een error met deze code omdat de console.log buiten de functie is.
    Omdat de variable in de functie wordt beschreven kan console.log niet bij deze variable en ziet niks om de 'definen'.

    Opdracht 2

    Kijkend naar de volgende code:

    let carName; function printCar() { carBrand = "Volvo"; } console.log(carBrand);

    Wat wordt er in de console gelogt?

    Dit is een voorbeeld van een global-scope. Dat zie je aan de plek waar let staat. Dit is buiten de funcite en dus kan je er bij ook al ben je niet bezig in de funcite.
    Je krijgt nu een error, omdat en in de funcite een variable wordt genoemdt dat niet bestaat. Hetzelfde geldt voor de variable dat genoemd wordt in console.log.

    Opdracht 3

    Opdracht bescKijkend naar de volgende code:hrijven

    function printCar() { carBrand = "Volvo"; } printCar(); console.log(carBrand); Wat wordt er in de console gelogt?

    Hier staat de variable in de functie, maar wordt zo genoemd buiten de functie, dat het wel toegankelijk is. Dit komt omdat de funtie - de funtie heeft gekregen om een variable te roepen.
    Dit zie je aan de de naam na de functie. Deze wordt weer hergebruikt buiten de functie om. Zo wordt de funtie getriggerd om de gedeclareerde 'carBrand' te roepen. En de console.log logt de gedeclareerde variable carBrand.

    Opdracht 4

    Kijkend naar de volgende code:

    function printCar() { carBrand = "Volvo"; return function(type) { console.log(carBrand + " " + type); } } const car = printCar(); car("v40");

    Wat wordt er in de console gelogt? Hoe is dit anders voorgaande opdrachten? Lees je meer over closures: Link

    Dit is een global scope. Dit kan je zien aan de const dat buiten de functie wordt beschreven als de type 'car'.
    De functie houdt in dat het de const variable en de functie hetzelfde is. In de funtie wordt beschreven wat het inhoudt.
